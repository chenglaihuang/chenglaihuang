# Code Review

- When asked to do code reviews, PLEASE:
  - Focus on bugs and concrete issues, do NOT just say the code is good or bad.
  - Comment on the exact line where changes need to be made.
  - Be concise, use point form if possible.
  - Annotate each comment with "nit:", "suggestion:", "bug:", "non-blocking:", "question:" etc where possible
  - If possible, use Github suggested fix feature to suggest what code change should be done.
